# claude-switch 홈 메뉴·다국어·rename 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** TUI를 홈 메뉴(로고+버전) 중심으로 개편하고 프로필 저장/이름변경 화면·다국어(en/ko)·버전 표기를 추가, GIF를 적정 비율로 재녹화.

**Architecture:** `internal/i18n`(키→en/ko 테이블, T()) 신설. tui.Model에 screen 상태 머신(home/profiles/save/rename/settings) 도입, 입력은 bubbles/v2 textinput. Backend 인터페이스에 Save/Rename/Language를 추가해 데모 모드도 동일 플로우 지원. app에 Rename 추가, CLI에 rename/lang/version 서브커맨드.

**Tech Stack:** 기존 + `charm.land/bubbles/v2`(textinput, v2.1.0 확인됨).

---

### Task 1: i18n 패키지

**Files:** Create `internal/i18n/i18n.go`, `internal/i18n/i18n_test.go`

- Lang("en"/"ko"), SetLang/Current/T(key). 키 누락 시 en 폴백, en도 없으면 key 반환
- messages: TUI(menu.*, home.*, profiles.help, confirm.delete, status.*, usage.loading, save.*, rename.*, settings.*) + CLI(cli.saved, cli.switched, cli.takesEffect, cli.deleted, cli.renamed, cli.noProfiles, cli.activeProfile, cli.langSet, usage 전문)
- 테스트: en/ko 키셋 동치(reflect로 두 맵 키 비교), 폴백 동작, fmt verb 개수 일치(% 개수 비교)
- 커밋: `feat: i18n 패키지 (en/ko)`

### Task 2: config.Language + app.Rename (TDD)

**Files:** Modify `internal/config/config.go`(Language 필드), `internal/app/app.go`, 테스트 추가

- config.Config에 `Language string \`json:"language,omitempty"\``
- app.Rename(old, new): old 존재·new 비공백·미중복 검사 → 키체인 Get(old)→Set(new)→Delete(old) → config 이름·Active 갱신·저장 → i18n 메시지 출력
- 테스트: 성공(키체인 항목 이동·Active 갱신), old 미존재, new 중복, 기존 회귀
- app의 성공 메시지(Fprintf)들을 i18n.T 키로 교체 (에러는 영어 유지)
- 커밋: `feat: 프로필 이름 변경(Rename) + config 언어 설정`

### Task 3: TUI 화면 상태 머신 (TDD)

**Files:** Modify `internal/tui/tui.go`, `internal/tui/view.go`, `internal/tui/demo.go`, 테스트 추가

- Backend에 Save(name)·Rename(old,new)·Language()·SetLanguage(l) 추가, appBackend/demoBackend/fakeBackend 구현
- Model: screen(home/profiles/save/rename/settings), menuCursor, settingsCursor, input(textinput), renameTarget, version
- Update: 입력 화면(save/rename)은 enter/esc만 직접 처리하고 나머지 키는 textinput.Update로 위임. ctrl+c 전역 종료. q는 home/profiles/settings에서 종료(입력 화면 제외)
- handleKey: home 메뉴 이동·선택(프로필 0개면 Save에 기본 커서·Manage는 안내), profiles에 n(rename 진입)·esc(home) 추가, settings 언어 토글·적용
- actionMsg에 goTo/fetch 필드 추가(저장 성공 → profiles 이동+fetchAll)
- View: 화면별 렌더링. 홈에 ASCII 로고(박스드로잉, 폭 ~38)+`v{version}`+메뉴. 모든 라벨 i18n.T
- 테스트(handleKey/submitInput 직접 호출): 홈 메뉴 이동→Manage 진입, 0개일 때 Save 기본 커서, save 제출→Backend.Save 호출+profiles 전환, rename 플로우, settings 언어 전환→SetLanguage 호출
- 커밋: `feat: 홈 메뉴·저장/이름변경 화면·설정 화면`
